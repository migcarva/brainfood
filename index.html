Homebrew

this is like apt-get or yum, but for mac

but... what does homebrew do?

it installs the stuff you need that Apple didn’t.


                    $ brew search wget
                

                    $ brew install wget

list all available packages

$ brew search | less

and can update them for you!


                    $ brew update
                

                    $ brew upgrade

update: update the formulae and Homebrew itself

upgrade: upgrades everything

brew cleanup + brew doctor

https://brew.sh/

                    $ brew tap caskroom/cask
                

                    $ brew cask search google-chrome
                

                    $ brew cask install google-chrome

Dotfiles is all about automating your setup process

  • clone / fork repo
  • tweak if needed
  • run script

fork a repo with your (or others) dotfiles

tweak them for more customization options, or to add personal or confidential details (like passwords, ssh..)

depending on the dotfiles, some have shell scripts to automate the process

My current setup


                    bash -c "$(curl -LsS https://raw.github.com/.../setup.sh)"

Mines are a fork of Alrra's dotfiles with some tweaking and personal options, like which programs to install, which macOS settings to set, etc...

I only need to run this command and I can go grab a coffee...

The setup process will:

  • Download the dotfiles on your computer
  • Create some additional directories
  • Symlink the git, shell, and vim files
  • Install applications / command-line tools
  • Set custom preferences
  • Install vim plugins

this is what happens!

this works both for windows, linux and mac

Although you need separate repos for each OS

just fork them!

styleguide and coding conventions

visual styleguide

This is good for us if we:

  • have developers of differing abilities and specialisms
  • have a number of different developers working on a product at any given time
  • on-board new staff regularly
  • have a number of codebases that developers dip in and out of

A good styleguide, when well followed, will:

  • set the standard for code quality across a codebase
  • promote consistency across codebases
  • give developers a feeling of familiarity across codebases
  • increase productivity
